# guide-mk-index

## 项目简介

guide-mk-index 是一个用于归档和发布多个独立 HTML 页面的仓库。它不针对任何特定域名或网站，而是作为一个通用的页面索引和展示空间，方便存放各类自包含的 HTML 资源。

本仓库的主要目的是提供一种结构化的方式来管理、版本化并公开这些页面，使其能够被直接访问或引用。

## 目录结构

仓库的典型目录结构如下：

```
guide-mk-index/
├── index.html          # 页面列表索引文件
├── pages/              # 存放独立 HTML 页面的目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
└── README.md           # 本文件
```

- **index.html**: 仓库的入口页面，通常列出所有归档页面的链接和简要描述。
- **pages/**: 存放所有独立 HTML 页面的主目录。每个页面应为一个完整的、自包含的 HTML 文件。

你可以根据需要调整目录结构，但建议保持 `pages/` 作为页面存放的核心位置。

## 页面归档说明

所有归档的 HTML 页面应满足以下基本要求：

- 页面应自包含，即所有样式和脚本内嵌在单个 HTML 文件中，避免外部依赖（除非特别说明）。
- 页面文件应放置在 `pages/` 目录下，文件名建议使用小写字母和连字符（如 `my-page.html`）。
- 每个页面应包含清晰的标题和内容，便于直接浏览或嵌入其他场景。
- 索引文件 `index.html` 应定期更新，以反映 `pages/` 目录中的最新页面列表。

### 添加新页面

1. 将新的 HTML 文件放入 `pages/` 目录。
2. 更新 `index.html`，在列表中添加指向新页面的链接和简要说明。
3. 提交更改并推送至仓库。

## 维护说明

- 本仓库由维护者定期更新，但欢迎任何形式的页面提交或建议。
- 如果发现页面无法正常访问、链接失效或内容有明显错误，请通过 Issues 反馈。
- 仓库不承诺对页面内容的长期可用性，但会尽力保持索引的准确性。
- 请勿在页面中添加任何形式的恶意代码、跟踪脚本或违反法律法规的内容。
